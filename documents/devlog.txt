Friday, 19 June 2026 Part 1 (F190626P1)
---------------------------------------
What we have accomplished from the start of the project until now:

1. Initial Setup & Architecture Constraints:
   - Configured the project to run Qwen3-TTS entirely on CPU (targeting Intel processors with an 8GB RAM limit) by completely disabling CUDA and flash_attention.
   - Decided to use the 0.6B parameters models (Qwen3-TTS-0.6B-Base and CustomVoice) to prevent Out-Of-Memory crashes.

2. Backend Development (FastAPI):
   - Created `model_manager.py` for thread-safe switching between the Base and CustomVoice PyTorch models in memory.
   - Created `audio_processor.py` to handle audio generation, bypassing garbage collection bottlenecks to improve CPU speed.
   - Integrated Whisper-tiny to act as a transcriber fallback.
   - Built a monolithic `main.py` containing endpoints for TTS, voice cloning, transcription, and model switching.

3. Frontend Development (React + Vite):
   - Built a dual-tab dashboard featuring "Custom Voice" (using predefined characters like Vivian, Dylan, etc.) and "Voice Clone" modes.
   - Implemented `useMicrophone` hook and `AudioVisualizer` to allow users to directly record 3-10s voice samples for cloning.
   - Built the `HistorySidebar` to persistently store generated audio (with download functionality).

4. Advanced Voice Clone Features:
   - Added an "Instant Clone" processing strategy (x-vector only mode) that completely bypasses the need for reference text.
   - Added automatic transcription: If reference text is required, Whisper-tiny automatically transcribes the uploaded reference audio.
   - Implemented a "Personal Voice Library" feature: Users can now name and permanently save a voice clone profile to a local library, which populates a dropdown for instant future usage.

5. Major Full-Stack Optimization & Modularization Refactor:
   - Backend: Sliced the monolithic `main.py` into clean FastAPI `routers/` (`tts.py`, `voice_library.py`, `models.py`).
   - Frontend: Extracted massive files like `VoiceCloneForm.tsx` and `App.tsx` into dozens of tiny, highly focused child components (`VoiceLibrarySelector`, `ReferenceAudioInput`, `ProcessingStrategyToggle`, `TTSGeneratorCard`).
   - Render Optimization: Heavily utilized `React.memo` to eliminate extreme re-rendering lag (e.g. typing text no longer causes the entire app or audio visualizer to re-render).

6. "Obsidian & Silk" Premium UI Overhaul & Polish:
   - Completely stripped out old CPU-heavy CSS template styles (massive animated aurora blobs, heavy glassmorphism blurs).
   - Replaced it with a hyper-optimized "Obsidian & Silk" lightweight minimal aesthetic featuring a crisp `#0A0A0A` background, pastel accents, and satisfying spring-physics bouncy button animations.
   - Added history deletion functionality (trash can icons).
   - Added UX polish: Character selection now automatically switches to the character's respective native language.

8. Hardware Telemetry & UX Fixes:
   - Implemented IndexedDB for permanent local storage of audio blobs, preventing history loss on browser restarts.
   - Added real-time tracking of RAM and peak CPU usage (`psutil`) during SSE generation, displaying as UI badges.
   - Refined the Studio Preset FX (from aggressive EQ/Gate to a smooth Compressor/Limiter chain) based on feedback.
   - Re-labeled "Fast Mode" to "Low RAM Mode" to avoid user confusion about PyTorch's dynamic quantization CPU penalty.
   - Eliminated catastrophic OS freezing during startup by removing `device_map="cpu"` and injecting `low_cpu_mem_usage=True` to bypass `accelerate` overhead.
   - Fixed a React empty `src` warning by conditionally passing `undefined` to audio elements during IndexedDB loading.

Saturday, 20 June 2026 Part 2 (SA200626P2) :
9. CPU Optimization (Speed & Memory):
   - Changed PyTorch model loading dtype from float32 to bfloat16 to halve memory footprint and significantly increase inference speed without any CPU overhead.
   - Reverted INT8 dynamic quantization default back to False. While INT8 saves RAM, the CPU casting overhead drastically slows down generation on basic Intel architectures. The "Low RAM Mode" toggle in the frontend will remain the only way to activate it.
   - Fixed a crash during startup ("cannot pickle 'dict_keys' object") by correctly applying dynamic quantization to the inner PyTorch `model.model` rather than the HuggingFace wrapper object when the frontend toggle is used.
   - Completely stripped out the INT8 "Low RAM Mode" toggle entirely after determining its performance penalty was too severe for practical use.
   - Fixed a fatal backend crash that would trigger when selecting the Studio Quality Preset (due to missing Compressor and Limiter imports from pedalboard).

10. Premium UX Features:
   - Built a "Saved Profiles" feature that lets users save and load favorite combinations of Speaker, Language, and Instructions directly to localStorage.
   - Created a "Merge & Download Batch" tool in the History Sidebar that fetches individual blob URLs and cleanly concatenates their WAV arrays into a single continuous audio file.
   - Refactored `AudioQueue` to natively route through Web Audio API's `AudioContext` and exposed an `AnalyserNode` to drive a real-time reactive `AudioVisualizer` waveform during playback.
   - Completely refactored the Batch Generation architecture: Moved processing out of backend background tasks and directly into a frontend async loop, allowing the UI to instantly populate the History Sidebar and provide live `Generating 2 of 5` progress feedback.
   - Replaced timestamp-based Batch IDs with clean 6-character random hex strings and styled the History Sidebar to visually distinguish batch folders using inset grouping and threaded indentation.
   - Fixed a fatal "echoing/repeating" audio artifact in Voice Clone mode by disabling sentence-chunking on the backend, allowing Qwen to execute a single seamless generation without repeated reference audio bleeding.
   - Tied the Aurora Audio Visualizer rendering to a strict `isPlaying` state monitor hooked directly into the `AudioQueue` to prevent empty boxes from persisting on screen.
   - Re-engineered the estimated time-to-completion algorithm with a dynamic multiplier that recalculates based on the processing overhead of the selected output codec (WAV vs FLAC vs OGG).
   - Polished the entire layout to a strict single-screen UI architecture by disabling global body scrolling, introducing independent column-based `overflow-y` internal scrolling, and designing a custom sleek global scrollbar to replace the browser default.
