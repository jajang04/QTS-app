# Architecture

## Overview
The Qwen-TTS Web App consists of a React 18 (Vite) frontend and a Python (FastAPI) backend. The backend executes Qwen-TTS locally on the CPU, specifically optimized for Intel architectures with `bfloat16` precision and SDP attention, avoiding INT8 casting overhead.

## Backend
- **Framework**: FastAPI (Uvicorn)
- **Models**: 
  - `Qwen/Qwen3-TTS-12Hz-0.6B-CustomVoice` (Used for style-based voice acting)
  - `Qwen/Qwen3-TTS-12Hz-0.6B-Base` (Used for Zero-Shot Voice Cloning and Cross-Lingual mimicking)
- **Model Management**: Singleton `ModelManager` orchestrates hot-swapping between Base and Custom models via garbage collection.
- **Inference Mode**: Local CPU (`device_map="cpu"`, `dtype=torch.bfloat16`)
- **Audio Processing Engine**:
  - `audio_processor.py` intercepts generated Neural Audio codes, dynamically applying `pedalboard` post-FX (Compressor, Limiter, Reverb, EQ, Noise Gate).
- **Endpoints**:
  - `GET /api/speakers`: Returns available zero-shot voice actors.
  - `POST /api/generate_custom_voice`: SSE (Server-Sent Events) endpoint that streams Base64 audio chunks back to the client as sentences are generated by `chunk_text(text)`.
  - `POST /api/generate_voice_clone`: SSE endpoint utilizing a single continuous block-generation for sample-accurate reference audio trimming, preventing audio duplication artifacts.
  - `POST /api/save_custom_voice`: Analyzes and processes reference audio to extract and serialize Prompt Embeddings into a stored `.pt` tensor file for immediate reuse.

## Frontend
- **Framework**: React 18, Vite, TypeScript
- **Design System**: Strictly single-screen viewport architecture (`100vh`, `overflow: hidden`) with localized column scrolling. Deep obsidian theme (`#0A0A0A`) with `var(--accent-mint)` styling and dynamic global webkit scrollbars.
- **Core Components**:
  - `AudioQueue` (useTTSApi.ts): Buffers incoming streaming SSE chunks, decodes them via `AudioContext`, and sequentially triggers playback without stuttering.
  - `HistorySidebar`: Uses IndexedDB via `localforage` to persistently store blobs, timestamps, and generation configurations. Groups Batch generations into distinctly styled folders.
  - `AudioVisualizer`: Binds to an `AnalyserNode` connected to the `AudioQueue` to render an Aurora-themed real-time frequency bar graph, conditionally mounted via strict `isPlaying` state monitoring.
  - `Batch Generation`: Orchestrated entirely client-side; splits text by newline, sequentially calls endpoints, and injects results directly into the History Sidebar upon completion.
  - `Merge & Download`: Decodes offline Blob arrays and recursively concatenates them using `OfflineAudioContext` for seamless file merging.
